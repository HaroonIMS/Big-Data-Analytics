1. Technical Proposal: Fraud Detection

Design and implement a scalable, real-time ML pipeline to detect fraudulent financial transactions. The system must flag suspicious transactions instantly to minimize loss for banks and customers. The final deliverable is a fully deployable predictive fraud detection model.

2. Dataset Selection & Documentation

11 Columns

Target Variable

isFraud

3. Pipeline Architecture

The fraud detection pipeline processes high-volume streaming transactions in real time. It includes:

  • Data ingestion from Kafka streams
  • S3/HDFS-based scalable storage
  • Batch & real-time ETL using PySpark
  • Feature Store for storing engineered features
  • Model hosting for real-time prediction
  • MongoDB for storing fraud outputs

4. Machine Learning Methodology

Pre-processing Strategy

  • Mean/median imputation for numerical data
  • One-Hot Encoding for categorical variable “type”
  • Error-based feature engineering
  • Min-Max Scaling

Algorithm Recommendation – XGBoost

  • Handles imbalance using scale_pos_weight
  • High Recall — critical for fraud detection
  • Efficient for tabular data

5. Implementation Plan

  • Load historical S3/HDFS data
  • Split train/test chronologically
  • Preprocess + feature engineering
  • Train & tune XGBoost
  • Deploy model for real-time scoring
  • Consume Kafka → score → store in MongoDB
  • Archive predictions for retraining
